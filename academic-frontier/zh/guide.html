工作流程

这套系统应该如何支持我们的研究判断

把工作逻辑收在这里，让首页保持简洁好用。

从论文工作面进入

首页应从方向选择直接进入论文工作面，把引文、摘要、DOI、期刊指标和本地下载动作集中到一张卡片里。

让订阅 PDF 留在本机

PDF 获取应留在这台机器上。开放获取文件可自动归档，需要出版社权限的内容则进入 OpenClaw 浏览器队列，而不是发布到 GitHub Pages。

把追踪变成研究杠杆

这个雷达不只是监控面板，还应该持续把前沿证据压缩成阅读顺序、综合总结和下一步真正可做的贡献方向。

日常前沿工作流

采集cron + 浏览器

检索 GitHub、论文源、实验室页面和观察名单；优先收集高信号候选，而不是机械抓取整个世界。

  • 原始来源命中
  • 标准化元数据
  • 候选 PDF 链接
整理Codex 总结

为条目打分、识别重复项、提取“为什么重要”的说明，并尽早把弱信号移出活跃通道。

  • 已评分卡片
  • 趋势证据
  • 团队监测更新
综合人工复核

把每日证据流转化为趋势判断、机会缺口和明确的下一步研究问题。

  • 趋势图谱
  • 洞察备忘录
  • 潜在贡献点
执行混合模式

把强信号转化为阅读队列、复现实验目标、想法笔记和现实可行的项目方向。

  • 下载队列
  • 议程清单
  • 项目种子
